# Cute Animated World Design Skill

## Intent

귀여운 애니메이션 세계관에 들어온 듯한 사용자 경험을 제공하는 디자인.
부드러운 색감, 둥근 형태, 생동감 있는 마이크로 인터랙션을 통해 감정적
몰입도를 높이고, 친근하고 안전하며 즐거운 분위기의 인터페이스를
구현한다. 캐주얼 게임, 캐릭터 기반 서비스, 키즈/엔터테인먼트 앱에 적합한
비주얼 아이덴티티를 목표로 한다.

## Style Rules

-   그림자는 소프트 섀도(blur 8--16px, opacity 10--20%)로 깊이감을
    표현한다\
-   border-radius는 크게 유지한다 (12--24px)\
-   색상은 고채도·파스텔 팔레트를 사용한다\
-   주요 컴포넌트는 카드형 레이아웃으로 분리한다\
-   타이포그래피는 둥글고 가독성 높은 산세리프를 사용한다\
-   여백은 넉넉하게 사용하여 답답함을 제거한다\
-   버튼은 볼륨감 있는 형태와 명확한 컬러 대비를 갖는다\
-   배지는 단색 필 + 아이콘 조합을 기본으로 한다\
-   포커스 링은 발광 효과(glow)로 명확히 표현한다\
-   호버 상태는 컬러 변화 + 미세한 스케일(1.02)로 반응성을 준다

## Tokens

-   color.primary: pink-400\
-   color.accent: sky-400\
-   color.success: mint-400\
-   color.warning: lemon-400\
-   color.error: rose-400\
-   radius.sm: 10px\
-   radius.md: 16px\
-   radius.lg: 24px\
-   space.section: 24px\
-   space.element: 12px

## Do

-   캐릭터, 아이콘, 일러스트를 적극 활용한다\
-   색상 대비를 명확히 하여 명랑한 리듬감을 만든다\
-   모든 주요 액션에 마이크로 애니메이션(150--300ms)을 적용한다\
-   컴포넌트 모서리를 둥글게 통일한다\
-   빈 공간에도 감정적 요소(패턴, 스티커, 미니 일러스트)를 고려한다

## Don't

-   무채색 위주의 비즈니스 팔레트 사용 금지\
-   각진 형태 위주의 컴포넌트 사용 금지\
-   텍스트만으로 구성된 딱딱한 레이아웃 금지\
-   애니메이션 없는 정적 UI 지양\
-   과도하게 작은 터치 영역 금지 (최소 44px 이상)

## Dark Mode

-   다크 모드에서도 파스텔 포인트 컬러를 유지한다\
-   배경은 차콜/퍼플 톤으로 부드럽게 어둡게 조정한다\
-   그림자는 다크 모드에서 글로우 중심으로 변경한다\
-   캐릭터와 일러스트는 대비를 강화한다\
-   호버/액티브 상태는 밝기 + 채도 상승으로 표현한다

## Notes

동화적이고 감정 중심의 디자인 시스템. 사용자가 '도구를 쓴다'기보다
'세계에 들어온다'는 인상을 받도록 설계한다. 기능적 효율보다 정서적
경험과 기억성을 우선하며, 브랜드 캐릭터와 애니메이션이 UI의 핵심 요소로
작동한다.
